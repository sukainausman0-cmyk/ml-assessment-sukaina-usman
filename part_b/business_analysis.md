B1. Problem Formulation

(a) In this case, the problem can be trated as a supervised machine learning task as the goal is to predict the number of items sold. The target variable is items_sold, since the company wants to maximise the sales value. The input feature can include store related factors like store size and location type,the type of promotion used, compition density, and time based features such as month or festival period. Since we are predicting numerical value, this is a regression problem. Regression is appropriate here because the output is continous rather than categorical.

(b) Using items sold is more realiable than total revenue because revenue can be influenced by pricing and discounts,which may not accuratly reflect the demand. For example,a heavy discount might reduce revenue but still increase the number of items sold.In contrast, items sold directly captures how coustomers respond to a promotion. This highlights an important lesson in machine learning i.e, the target variable should actually align with the business goal to ensure meaningful results.

(c) Using a single global model for all 50 stores may not be effective because stores in diffrent locations can behave very diffrently. For instance, urban stores may have higher number of customers visiting the store and different customer preferance compared to rural stores. A better approch would be either build sperate stores for different model segments (such as urban, semi-urban, rural) or ensure that location based features are properly captured in the model. This helps the model account for variations across stores and improves prediction accuracy.

B3. Model Evaluation and Deployment 

(a) Since the data is time-based and span multiple years, the train-split test should be done chronologically, using earliar data from training and more recent data fro testing.For example the first two years of data can be used for training and the last year for testing. A random split is not appropriate for this case because it mixes the past and future data, leading to data leakge and unrealistic model performance. For evaluations such as MAE (Mean Absolute Error) and RMSE (Root Mean Squared Error) can be used. MAE provides the average error between predicted and actual values, while RMSE gives more weight to larger errors, making it useful for identifying significant prediction mistake in business context.

(b) Feature model can be used to understand why the model recommends different promotions for the same store in different months. Factors such as seasonality, feativals and customer behaviour patterns can vary across months and influence sales outcomes.For example December may have higher demand due to festive periods, making certain promotion more effective, while march may reflect regular buying behaviour. By analysing feature importance, we can identify which factors influenced the model's decision and explain these insights clearly to the marketing team, helping them trust and understand the recommendations.
